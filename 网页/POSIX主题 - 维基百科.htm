POSIX线程，通常被称为并行线程，是一种执行模型，从一种语言，以及并行执行模型独立地存在。它允许程序控制与时间重叠的多个不同的工作流程。每个工作流程都被称为线程，并通过调用POSIX Threads API来实现对这些流程的创建和控制。POSIX线程是由标准POSIX .1c，线程扩展（IEEE Std 1003.1c-1995）定义的API。

API的实现可以在诸如FreeBSD，NetBSD，OpenBSD，Linux，Mac OS X，Android [1]和Solaris之类的许多符合Unix的 POSIX操作系统上使用，通常捆绑为库libpthread。还存在DR-DOS和Microsoft Windows实现：在提供多个POSIX API的本机实现的SFU / SUA子系统中，以及第三方软件包（如pthreads-w32）中，

内容[ 编辑]

Pthreads定义了一组C编程语言类型，函数和常量。它使用pthread.h标题和线程库实现。

有大约100个Pthread程序，所有的前缀pthread_都可以分为四组：

  • 线程管理 - 创建，加入线程等
  • 互斥
  • 条件变量
  • 使用读/写锁和屏障的线程之间的同步

POSIX 信号量 API与POSIX线程一起工作，但不是在POSIX.1b（IEEE Std 1003.1b-1993）标准中定义的线程标准的一部分。因此，信号量程序的前缀是sem_替代pthread_。

示例[ 编辑]

说明在C中使用Pthreads的例子：

#include  <pthread.h> 
#include  <stdio.h> 
#include  <stdlib.h> 
#include  <assert.h>
 
#define NUM_THREADS 5
 
void *  perform_work （ void *  argument  ）
{ 
  int  passed_in_value ;
 
  passed_in_value  =  * （ （ int *  ）参数 ）; 
  printf （ “Hello World！It's me，thread with argument％d！\ n ” ， passed_in_value  ）;
 
  / *可选：在这里插入更多有用的东西* /
 
  返回 NULL ; 
}
 
int  main （ int  argc ， char **  argv  ）
{ 
  pthread_t  threads [  NUM_THREADS  ]; 
  int  thread_args [  NUM_THREADS  ]; 
  int  result_code ; 
  无符号 索引;
 
  //创建的所有线程逐一
  对（ 索引 =  0 ;  索引 <  NUM_THREADS ;  ++ 索引 ）
  { 
    thread_args [  索引 ]  =  指数; 
    printf （“主要：创建线程％d \ n ” ， 索引）; 
    result_code  =  pthread_create （ ＆threads [ index ]， NULL ， perform_work ， ＆thread_args [ index ]  ）; 
    assert （ ！result_code  ）; 
  }
 
  //等待每个线程完成
  for （ index  =  0 ;  index  <  NUM_THREADS ;  ++ index  ）
  { 
    //块直到线程'index'完成
    result_code  =  pthread_join （ threads [  index  ]， NULL  ）; 
    assert （ ！result_code  ）; 
    printf （ “主要：线程％d已经完成\ n ” ， 索引 ）; 
   }
 
   printf （ “主要：所有线程成功完成\ n ”  ）; 
   退出（ EXIT_SUCCESS  ）; 
}

该程序创建五个线程，每个执行函数perform_work，将该线程的唯一编号打印到标准输出。如果程序员希望线程相互通信，这将需要在任何函数的范围之外定义变量，使其成为全局变量。该程序可以使用gcc编译器使用以下命令进行编译：

gcc pthreads_demo.c -pthread -o pthreads_demo

用于Windows的POSIX主题[ 编辑]

Windows本身不支持pthreads标准，因此Pthreads-w32项目旨在提供一种便携式和开源的包装器实现。它也可以用于将Windows平台的Unix软件（使用pthreads）进行很少或没有修改。[3]使用一些补丁程序，最新版本2.8.0与64位Windows系统兼容。[4] [5] [6] 2.9.0据说也是64位兼容的。[7]

mingw-w64项目还包含pthreads，winpthreads [8]的包装器实现，它尝试使用比Pthreads-w32项目更多的本机系统调用。[9]

适用于UNIX / Subsystem（用于基于UNIX的应用程序）的Windows Services中的Interix环境子系统提供pthreads API的本机端口，即不映射到Win32 / Win64 API，而是直接构建在操作系统syscall界面上。[10]
